# デジタル・ホワイトペーパー開発標準マニュアル

## 1. プロジェクト概要
高品質なWebベースのホワイトペーパーを制作し、GitHub/Vercelを通じて公開、さらにPDF出力（印刷）に最適化するための開発要件です。

## 2. 要件定義（マーケティング・ライティング）
*   **タイトル構造**: 「ターゲット」+「具体的数値（1億円等）」+「ロードマップ/戦略」を含める。
*   **トーン＆マナー**: 
    *   「です・ます」調で統一。
    *   専門用語（横文字）を避け、直感的に理解できる日本語を使用。
    *   独自の比喩（例：「静寂の谷」）を用いて、読者の共感と権威性を構築。
*   **構成要素**:
    *   序章（課題提起）
    *   本編（具体的な戦術・数値データ）
    *   TIPS（実務に役立つ具体的なDX手法等）
    *   CTA（個別相談・会員登録への誘導、QRコード設置）

## 3. 技術仕様（インフラ・デプロイ）
*   **プラットフォーム**: Vercel
*   **ソース管理**: GitHub (Branch: `master` または `main`)
*   **ディレクトリ構成**:
    *   `index.html`: リダイレクトまたはメインコンテンツの入り口。
    *   `whitepaper_part1.html`: メインコンテンツファイル。
    *   `css/style.css`: 外部スタイルシート。
    *   `vercel.json`: 配信設定ファイル。
*   **環境構築の注意点**: 
    *   OneDrive等の同期遅延を避けるため、ローカルディスク（`C:\projects\`等）で作業する。

## 4. PDF・印刷最適化（CSS）
WebコンテンツをPDFとして正しく出力するための必須設定です。

```css
@media print {
  /* ヘッダー・フッターの非表示（ブラウザ標準情報の抑制） */
  @page { margin: 10mm; }
  
  /* コンテンツの改ページ制御 */
  .page-break {
    page-break-before: always;
  }
  
  .no-break {
    page-break-inside: avoid;
    break-inside: avoid;
  }
  
  /* 不要な要素の非表示 */
  .no-print { display: none !important; }
}
```

## 5. Vercel 設定 (`vercel.json`)
静的HTMLサイトを確実にホストするための設定です。

```json
{
  "buildCommand": "",
  "outputDirectory": ".",
  "routes": [
    { "src": "/", "dest": "/whitepaper_part1.html" },
    { "src": "/(.*)", "dest": "/$1" }
  ]
}
```

## 6. AIへの指示（プロンプト）テンプレート
他のプロジェクトでAIに指示を出す際の推奨文言です。

> あなたは世界一のライター、マーケター、およびフロントエンドエンジニアです。
> 以下の条件でホワイトペーパーを制作してください。
> 1. タイトルは「[ターゲット]向けの[成果]ロードマップ」とする。
> 2. Webでの閲覧とPDF印刷（A4サイズ）の両方を完璧にこなすデザインにする。
> 3. @media printを使用して、コンテンツが途中で切れないように改ページを制御する。
> 4. 最後には[個別相談URL]へのCTAとQRコードを配置する。
> 5. 技術的にはGitHubにPushしてVercelで自動デプロイする構成にする。
