# デジタル・ホワイトペーパー開発標準マニュアル（完全再現版）

## 1. プロジェクト概要
本マニュアルは、高品質なWebベースのホワイトペーパーを量産し、GitHub/Vercelで自動公開、さらにPDF（A4印刷）として完璧なレイアウトを維持するための標準規格です。

## 2. ディレクトリ構成（マルチプロジェクト対応）
プロジェクトが増えても管理・拡張が容易なフォルダ構造を採用します。

```text
whitepaper_local/
├── vercel.json           # 全体配信設定
├── reproduction_guide.md # 本マニュアル
├── index.html            # メインURLへのリダイレクト
│
├── 01_project_name/      # 各プロジェクトフォルダ
│   ├── index.html        # コンテンツ本体
│   ├── css/              # プロジェクト専用CSS
│   └── public/           # 画像・ロゴ・QRコード等
│
└── [02_...]              # 次のプロジェクトも同様の構造
```

## 3. デザイン定義（UI/UX・ブランディング）
「世界最高峰」の信頼感を演出するため、以下のデザイン・ルールを厳守します。

*   **タイポグラフィ**: `Noto Sans JP` を使用。本文は 13.5px、行間 1.8〜1.9 で可読性を最大化。
*   **カラーパレット**:
    *   **メイン（Navy）**: `#1a365d`（信頼感、専門性）
    *   **ソフト（Background）**: `#F5F7FB`（清潔感）
    *   **警告・注意（Alert/Warning）**: `#D93025` / `#F9AB00`（リスク喚起）
    *   **成功（Success）**: `#0F9D58`（ポジティブな結果、CTA）
*   **視覚コンポーネント**:
    *   **Callout（注釈）**: アイコンと背景色を使い、重要なポイントを強調。
    *   **Summary Table**: ネイビーのヘッダーを用いた見やすい表。
    *   **Checklist**: チェックボックス付きリストで実務的な点検を促す。
    *   **Disclaimer Box**: 最終ページに配置する、法的リスクを回避するための厳格な免責事項。

## 4. PDF・印刷最適化仕様
ブラウザの「PDFとして保存」で、そのまま配布用資料になるための設定です。

*   **ページ基本設定**: A4縦 (`210mm x 297mm`)。
*   **改ページ制御**:
    *   `.page`: ページごとのコンテナを作成し、`page-break-after: always` を指定。
    *   `page-break-inside: avoid`: 重要なBOXやテーブルがページをまたいで切れないように制御。
*   **非表示設定**: 印刷時に不要なWebパーツ（ナビゲーション等）を `.no-print` で消去。

## 5. インフラ設定 (`vercel.json`)
ルートディレクトリに配置し、複数プロジェクトのURLを自動構成します。

```json
{
  "buildCommand": "",
  "outputDirectory": ".",
  "routes": [
    { "src": "/", "dest": "/01_nursing_roadmap/index.html" },
    { "src": "/(.*)", "dest": "/$1" }
  ]
}
```
